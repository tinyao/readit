    OpenAI工程师团队用五个月时间完成了一项实验：构建并交付一款完全由Codex AI智能体编写代码的软件产品，没有任何一行代码由人工编写，估计仅用了手工编写所需时间的约十分之一。团队由三名工程师扩展至七名，共合并约1500个Pull Request，代码库达到约一百万行，涵盖应用逻辑、测试、CI配置、文档和内部工具等。在此过程中，人类工程师的角色转变为系统设计者和任务协调者，专注于拆解目标、构建反馈回路、管理知识库，并将应用的可观测性（日志、指标、UI驱动等）直接暴露给Codex以提升其自主执行能力。团队将代码仓库作为唯一的记录系统，通过结构化文档目录、严格的分层架构约束和自定义linter来确保智能体生成代码的一致性和可维护性。这一实践表明，在智能体优先的工程环境中，速度的关键不在于让AI更努力，而在于为其提供清晰的环境、工具和可强制执行的约束。

    本文探讨了AI智能体的快速发展及其对未来软件行业的深远影响。作者指出，智能体正从简单的聊天机器人演变为拥有独立计算环境、能执行复杂任务的自主系统，并将渗透到合同审查、客服、财务审计、医学研究等几乎所有经济领域。未来企业中智能体数量将远超人类员工，成为软件的主要用户，这要求所有软件必须以API为核心进行重构，以满足智能体的需求。文章还指出，传统的按席位收费商业模式将难以适应智能体时代，软件厂商需要引入基于消费量或使用量的定价模型。此外，围绕智能体的全新基础设施、工具和平台生态也将随之兴起，形成全新的技术格局。
